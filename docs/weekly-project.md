# Weekly Project Calendar Initiative

## Overview

To encourage hands-on learning and consistent engagement, support.pschi02.com will begin following a structured weekly project calendar.

The goal is simple: introduce a new project idea every Monday, work through the implementation during the week, document progress along the way, and publish the completed project by the weekend.

This approach helps students build practical skills while creating a growing portfolio of documented work.

## Why This Change Was Made

One of the biggest challenges students face is deciding what to work on next.

Many learners complete labs and assignments but struggle to find real-world projects that help reinforce concepts and demonstrate skills to employers.

By introducing a new project idea each week, students can:

* Practice hands-on technical skills
* Learn project planning and execution
* Develop documentation habits
* Build portfolio-ready projects
* Create a consistent learning routine

## Weekly Workflow

### Monday – Project Launch

A new project idea will be published to the Knowledge Base.

The project will include:

* Objectives
* Required tools
* Expected outcomes
* Suggested learning goals

A LinkedIn post will also be published introducing the week's project.

### Tuesday – Thursday – Project Development

Students work on the project while documenting:

* Installation steps
* Configuration changes
* Challenges encountered
* Troubleshooting performed
* Lessons learned

Progress updates may be shared through LinkedIn and GitHub.

### Friday – Saturday – Documentation & Review

The project documentation is finalized and added to GitHub.

Supporting materials may include:

* Screenshots
* Configuration files
* Architecture diagrams
* Lessons learned
* Validation testing

### Sunday – Project Wrap-Up

The completed project is reviewed and published.

Students are encouraged to reflect on:

* What they learned
* What worked well
* What could be improved
* Future enhancements

## Benefits

This weekly schedule provides several advantages:

* Consistent content for the Knowledge Base
* Regular LinkedIn updates
* Ongoing GitHub portfolio development
* Continuous hands-on learning
* Increased community engagement

Rather than waiting months to complete a large project, students can finish smaller projects regularly and build momentum.

## Long-Term Goal

The long-term vision is to create a growing library of practical IT, Linux, cloud, cybersecurity, monitoring, networking, and help desk projects that students can follow and adapt for their own learning.

Over time, this initiative will create a documented archive of real-world projects while encouraging students to learn by building.

## Conclusion

Starting a new project every Monday creates a predictable rhythm for learning, documentation, and community engagement.

By combining hands-on practice with regular documentation and public progress updates, students can develop both technical skills and professional habits that are valuable throughout their careers.
